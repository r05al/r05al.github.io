Ruby classes vs. JavaScript constructor functions

"A rose by any other name would smell as sweet"

Aug 24, 2015

Ruby classes and JavaScript constructor functions are essentially the same idea and concepts, just dressed differently. I will walk you through an example of each and show how they differ.

In Ruby, we define a class by stating class followed by the name we choose, typically capitalized. We then use the method initialize to create an instance of this class passing arguments that we want to assign to this instance. Within the class we also define any methods we would call on our instance of this class, to define our objects main purpose.

In JavaScript we define an Object constructor function starting with function followed by any allowable variable name, then parentheses to enclose the arguments that we will pass in. In JavaScript we take these arguments and assign them to variable Object that we create using the this. and the name of the argument set to equal the argument. Arguments can be strings, numbers, boolean, functions, other objects, etc. 

    class BlogPost

    	attr_reader :name, :date, :content

    	def initialize(name, date, content)
    		@name = name
    		@date = date
    		@content = content
    	end

    	def print
    		puts "#{name} - #{date}"
    		puts "#{content}"
    	end
    end

    #CREATES INSTANCE OF CLASS
    pete = BlogPost.new("Today's Post", "Aug 24", "Hello world!")

    pete.print 				#CALLS METHOD

    Today's Post - Aug 24 	#OUTPUT
	Hello world! 			#OUTPUT
	nil

Here is the same example in Javascript. 

 
	function BlogPost(name, date, content) {
	        this.name = name;
	        this.date = date;
	        this.content = content;
	        this.print = print;
	    };

	function print() {
	  console.log(this.name + " - " + this.date);
	  console.log(this.content);
	}
	  
	//CREATES INSTANCE OF OBJECT 
	var pete = new BlogPost("Today's Post", "Aug 24", "Hello world!");

	pete.print();			//CALLS FUNCTION

	Today's Post-Aug 24 	//OUTPUT
	Hello world! 			//OUTPUT

Creating an instance of a class in Ruby vs creating an object using the constructor function are similiar, in that you pass the arguments you want to assign and you indicate that it is "new". In Ruby the method is defined in the class. We can do the same by defining the function within the constructor. Above we defined it globally, so I can be shared with other objects. In Ruby we can do the same, but the method would have to be recoded to call the instance variables differently.

All in all, both get the job done. Preference will be based on the context of your project and the scope of how your objects will communicate. I am not that familiar with other languages, but I'm sure they have similiar things that emulate these ideas into their syntax. Keep exploring readers!
