Ruby Classes

Declassified

Aug 10, 2015

Ruby allows programmers to create related objects that allow shared behavior through the use of classes. With classes we are able to create a shared collection of methods. Methods defined in the class are made to be accessible to all instances of the class. These are called instance methods. We can use these methods to assign values to variables of these particular instances. These are called instance variables. They always start with an @ (at sign), are only visible to that class, and able to be called in any of the class's methods. Here is a quick example where we create the class Dog, pass in the name and breed to the initialize method. Initialize is executed when you send the message new to the class you have created. 


      class Dog
        def initialize(name, breed)
          @name = name
          @breed = breed
        end
      end

      fido = Dog.new("fido", "Terrier")

Typically we don't want to rename our dog, but let's say he's a new pup we have adopted. Since he's a pup he's age 0. 

       class Dog
        attr_reader :name 

        def initialize(name, breed)
          @name = name
          @breed = breed
        end

        def breed
          @breed
        end

        def new_name(name)
          @name = name
        end


        def age=(years)
          @age = years
        end
      end     

      fido = Dog.new("Fido", "Terrier")
      fido.new_name("Rex")
      fido.age=(0)

      p fido.name
      p fido.breed

Here are a few new additions. We added attr_reader :name. This allows us to call Fido, ahem, Rex by using method name. This is called object attribute reader, a getter method. We are able to read the instance variable @name through this. We can also write through an attribute writer. We defined the setter method for age incorporating the equals sign. Doing so, in order to call this method we must mirror that using fido.age=(0). This sets the age, hence setter. We can also rewrite this setter method to attr_writer :age. It enables writing to the instance variable @age but not reading. Meaning, we cannot get the age by calling fido.age. If we used attr_accessor :age THEN we could read and write the age of our dog.

Create your own classes, add your own methods and assign your own variables!
