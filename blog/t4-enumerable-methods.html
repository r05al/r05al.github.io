Enumerable#cycle

cycling

Aug 3, 2015

First things first, an Enumerable is a module that gives classes with the each method access to a number of collection-related behaviors. One of those behaviors we will talk about is Enumerable#cycle. Enumerable#cycle yields all the elements of the collection in a loop. In the below it will run n times through, where if you do not provide an argument, it will run infinitely. If the collection is empty or the argument passed is 0 or less, it will not execute. The return is nil after regular execution.

sample_collection.cycle(n)

We are able to pass a code block to each element of the collection. This will be saved to an internal array so it is not destructive. If there is no block given, this returns an enumerator. We can store this enumerable in a variable then chain it on to other Enumerable methods to combine things, such as .reverse_each.cycle to return in reverse. You can also call .next on your enumerable to move forward to the next element in your collection.

Play around with using Enumerable methods on your collections. Try chaining two together and think about the logic behind the combination. Happy coding.
